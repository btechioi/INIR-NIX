# Dotfiles Deployment for iNiR HM Module — Implementation Plan

**Goal:** Extend the iNiR Home Manager module to deploy the full set of dotfiles (from `dots/.config/` and `defaults/`), the QML runtime symlink, state initialization, wallpaper deployment, and session environment variables — matching what `./setup install` does imperatively.

**Architecture:** Hybrid approach using HM's `xdg.configFile` (for dotfiles), `home.file` (for runtime symlink), `home.activation` (for state dirs/files/wallpapers), and `home.sessionVariables` (for env vars). All changes go into `nix/home-module.nix` since they belong in the existing `config = mkIf cfg.enable { ... }` block.

**Design:** `thoughts/shared/designs/2026-06-03-inir-nix-dotfiles-deployment-design.md`

## Design Decisions & Gap Fills

1. **`recursive` for `home.file` symlink**: Design says `recursive = true`. However, with `home.file`, `recursive = true` creates individual symlinks for each file INSIDE a directory source. For the QML runtime at `~/.config/quickshell/inir`, we want a single directory symlink to the store path. **Decision**: Omit `recursive` (defaults to `false`), creating a single symlink `~/.config/quickshell/inir → /nix/store/.../share/quickshell/inir`.

2. **`defaults/config.json` skipped**: Design maps it to `inir/config.json`, but the module already generates `inir/config.json` via `inirConfigGen.generate`. Adding a static source entry for the same path would conflict. **Decision**: Skip `defaults/config.json` — the generated config is authoritative.

3. **Merge order in `mkMerge`**: New source-based dotfiles go as the 3rd element (after generated configs and palette dotfiles). This means palette-based templates (when nix-colors is available) override the static defaults, which is the correct behavior — palette-aware configs are better.

4. **`dots/.config/niri/config.kdl` skipped**: Already generated by `niriConfigGen.generate` — adding a static copy would conflict.

5. **Activation DAG ordering**: `writeBoundary` → `createStateDirs` → `createStateFiles` → `deployWallpapers`. Ensures dirs exist before files are created, and wallpaper copy happens last.

#### A — Runtime Payload Symlink

Insert **after line 170** (`home.packages = [ inirPkg ];`):

```nix
    # Runtime QML payload symlink — makes the shell runtime available
    # at the path Quickshell expects (~/.config/quickshell/inir)
```

**Design justification:** The QML runtime files live in the Nix store at `$out/share/quickshell/inir/`. The shell expects them at `~/.config/quickshell/inir/`. A `home.file` symlink bridges this without copying.

#### C — State Initialization & Wallpaper Deployment (`home.activation`)

Insert **after** the closing `];` of `xdg.configFile` (after line 203), **before** `programs.niri` (line 205):

```nix
    # State initialization: create dirs and files the shell expects at startup
    home.activation = {
      # Create state directories for generated content
      createStateDirs = lib.hm.dag.entryAfter ["writeBoundary"] ''
        mkdir -p "${config.xdg.stateHome}/quickshell/user/generated/wallpaper" || true
        mkdir -p "${config.xdg.stateHome}/quickshell/user/generated/terminal" || true
      '';
      # Initialize empty state files
      createStateFiles = lib.hm.dag.entryAfter ["createStateDirs"] ''
        touch "${config.xdg.stateHome}/quickshell/user/generated/gamemode_active" || true
        touch "${config.xdg.stateHome}/quickshell/user/generated/notepad.txt" || true
        echo '[]' > "${config.xdg.stateHome}/quickshell/user/generated/todo.json" || true
        echo '[]' > "${config.xdg.stateHome}/quickshell/user/generated/notifications.json" || true
      '';
      # Deploy bundled wallpapers to ~/Pictures/Wallpapers/inir/
      # Only copies if target dir is empty (avoids overwriting user's collection)
      deployWallpapers = lib.hm.dag.entryAfter ["createStateFiles"] ''
        mkdir -p "${config.home.homeDirectory}/Pictures/Wallpapers/inir"
        if [ -z "$(ls -A "${config.home.homeDirectory}/Pictures/Wallpapers/inir" 2>/dev/null)" ]; then
          cp -a "${inirPkg}/share/inir/wallpapers/." "${config.home.homeDirectory}/Pictures/Wallpapers/inir/" || true
        fi
      '';
    };
```

**Error handling:** All commands use `|| true` except the mkdir + ls guard for wallpapers. State files are non-critical — failures won't block HM activation. Wallpaper copy only runs if the target dir is empty (determined by `ls -A` returning empty).

```nix
{ lib, config, pkgs, inputs ? { }, ... }:
let
  cfg = config.programs.inir;
  inirPkg = pkgs.callPackage ./package.nix { };

  colorPipeline = import ./color-pipeline.nix { inherit lib inputs; };

  # Generate a KDL config file for niri from the options tree
  generatedNiriConfig = niriConfigGen.generate (lib.filterAttrsRecursive (n: _: n != "_module" && n != "_type") cfg.niri);

          options = {
            action = lib.mkOption {
              type = lib.types.str;
              description = "Action to perform (spawn, close-window, etc.)";

              type = lib.types.listOf lib.types.str;
              default = [ ];
              description = "Arguments to the action";

            "Mod+Return" = { action = "spawn"; args = [ "alacritty" ]; };

          }
        '';
        description = "Niri keybindings";

      type = lib.types.nullOr lib.types.str;

      type = lib.types.nullOr lib.types.str;

    inputs.niri-flake.homeManagerModules.niri

  config = lib.mkIf cfg.enable {

      # ── NEW: Source-based dotfiles (always deployed) ──────────────────

        "fish/config.fish".source = ./../dots/.config/fish/config.fish;
        "fish/auto-Niri.fish".source = ./../dots/.config/fish/auto-Niri.fish;
        "mpv/mpv.conf".source = ./../dots/.config/mpv/mpv.conf;
```
